## 🚀 Getting Started

### Prerequisites

- **Bun** >= 1.0 ([install](https://bun.sh/docs/installation))
- **Node.js** >= 18 (for some tooling)
- **PostgreSQL** >= 14 (for production only — dev uses SQLite)

### Install & Run

```bash
# 1. Clone the repo
git clone <your-repo-url> freedom-cosmetic-shop
cd freedom-cosmetic-shop

# 2. Install dependencies
bun install

# 3. Set up environment variables
cp .env.example .env
# Edit .env with your values (or use defaults for dev)

# 4. Initialize the database
bun run db:push       # Creates SQLite DB + tables
bun run db:seed       # Seeds 3 categories + 18 products

# 5. Start the dev server
bun run dev
# → Open http://localhost:3000 in your browser
```

### First-time setup checklist

- [ ] `bun install` succeeds
- [ ] `.env` file created from `.env.example`
- [ ] `bun run db:push` runs without errors
- [ ] `bun run db:seed` shows "✅ Seed complete!"
- [ ] `bun run dev` starts the server
- [ ] `http://localhost:3000` loads the home page with products
- [ ] Add an item to cart → checkout → see confirmation page
- [ ] Click "Admin" in header → see the order you just placed

## 🔐 Environment Variables

All variables are documented in [`.env.example`](./.env.example). The most important ones:

### Required for dev (already set)

| Variable       | Default               | Description              |
| -------------- | --------------------- | ------------------------ |
| `DATABASE_URL` | `file:./db/custom.db` | SQLite connection string |
| `NODE_ENV`     | `development`         | Environment              |

### Required for production

| Variable                | Description                                            |
| ----------------------- | ------------------------------------------------------ |
| `DATABASE_URL`          | PostgreSQL connection string                           |
| `NEXTAUTH_SECRET`       | Random 32+ char string (use `openssl rand -base64 32`) |
| `PAYPACK_CLIENT_ID`     | PayPack client ID (for MTN MoMo)                       |
| `PAYPACK_CLIENT_SECRET` | PayPack client secret                                  |
| `CLOUDINARY_*`          | Cloudinary credentials (for image uploads)             |
| `AT_API_KEY`            | Africa's Talking API key (for SMS)                     |

### Feature flags

| Flag                         | Default | When to enable              |
| ---------------------------- | ------- | --------------------------- |
| `ENABLE_SMS_NOTIFICATIONS`   | `false` | After AT_API_KEY is set     |
| `ENABLE_EMAIL_NOTIFICATIONS` | `false` | After RESEND_API_KEY is set |
| `ENABLE_REAL_PAYMENTS`       | `false` | After PayPack is configured |
| `ENABLE_SEARCH_INDEXING`     | `false` | After Algolia is configured |
| `ENABLE_REDIS_CACHE`         | `false` | After Redis is configured   |

## 🗄 Database

### Dev (SQLite) — current

```bash
bun run db:push      # Push schema to SQLite
bun run db:seed      # Seed demo data
bun run db:studio    # Open Prisma Studio GUI at http://localhost:5555
```

### Production (PostgreSQL) — swap when ready

```bash
# 1. Install pg adapter
bun add pg
bun add -d @types/pg

# 2. Replace prisma/schema.prisma with prisma/schema.postgres.prisma
cp prisma/schema.postgres.prisma prisma/schema.prisma

# 3. Set DATABASE_URL in .env to your PostgreSQL connection string
# DATABASE_URL=postgresql://user:pass@host:5432/ubumwe?schema=public

# 4. Push the schema
bun run db:push

# 5. Seed
bun run db:seed
```

### Prisma schema highlights

- **Category** — `id`, `name`, `slug`, `description`, `image`
- **Product** — `price` (Int RWF), `compareAt`, `images` (JSON array), `rating`, `stock`, `featured`, `brand`
- **Order** — `orderNumber` (sequential, `UB-YYYY-NNNNN`), `paymentMethod`, `paymentStatus`, `status`, `subtotal`/`deliveryFee`/`total`
- **OrderItem** — Snapshot of product at purchase time (preserves order history even if product is deleted)

## 🌍 Deployment

### Frontend → Vercel

1. Push the repo to GitHub
2. Go to [vercel.com](https://vercel.com) → New Project → Import the repo
3. Add all env vars from `.env.example` in the Vercel dashboard
4. Set `DATABASE_URL` to your PostgreSQL connection string
5. Deploy — Vercel auto-detects Next.js

### Backend (if split) → Railway

The MVP keeps everything in Next.js (API routes). If you split into a separate Express backend later:

1. Go to [railway.app](https://railway.app) → New Project → Deploy from repo
2. Add a `backend/` directory with Express + the same Prisma schema
3. Set env vars in Railway dashboard
4. Railway auto-deploys on push to `main`

### Database → Supabase / Neon / Railway

For managed PostgreSQL with free tier:

- [Supabase](https://supabase.com/) — 500MB free, generous free tier
- [Neon](https://neon.tech/) — Serverless PostgreSQL, 3GB free
- [Railway](https://railway.app/) — Simple, $5/mo credit monthly

### Post-deploy checklist

- [ ] Set `NODE_ENV=production`
- [ ] Set `APP_URL` to your Vercel domain
- [ ] Set `NEXTAUTH_URL` to your Vercel domain
- [ ] Generate a strong `NEXTAUTH_SECRET`
- [ ] Configure PayPack webhook URL → `https://your-domain.com/api/webhooks/paypack`
- [ ] Configure Flutterwave webhook URL → `https://your-domain.com/api/webhooks/flutterwave`
- [ ] Set `ENABLE_REAL_PAYMENTS=true`
- [ ] Set `ENABLE_SMS_NOTIFICATIONS=true` (after testing)
- [ ] Verify Cloudinary upload preset
- [ ] Run `bun run db:seed` on production DB (if you want demo data)
